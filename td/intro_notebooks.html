1. Objectif

L'objectif de cette première activité est de se familiariser avec le concept des notebooks et les environnements de développement collaboratif.

2. Les Notebooks

Un notebooks en programmation est une interface permettant de développer des programmes (ici en Python) et d'en présenter les résultats de manière interactive en y incluant de la documentation. Cette manière de programmer met en pratique la démarche de programmation lettrée (literate programming, en anglais). Cette démarche est particulièrement bien adaptée aux projets de science des données qui mêlent en général développements informatiques, visualisations et interprétations.

Un notebook est donc constitué de blocs de code Python, des résultats associés (qui peuvent être des tableaux, des graphiques, du texte, etc) et du texte formaté dans un même document. L'objectif est de favoriser la transparence, la compréhension, la reproductibilité et le partage de travaux scientifiques et techniques.

L'utilisation de notebooks est désormais une pratique classique dans les projets de science des données dans le monde académique et en entreprises. Bien que dans ce cours, nous n'utiliserons que des notebooks Python, de nombreux outils proposent de travailler avec les principaux langages scientifiques usuels aujourd'hui (e.g. R, Ruby, Julia, etc).

3. L'environnement Deepnote

Dans ce module, nous travaillerons essentiellement avec la plateforme Deepnote (https://deepnote.com/). Il s'agit d'une plateforme de développement collaboratif gratuite permettant de développer en Python.

Travail à réaliser : Créer un premier notebook

  • Se rendre à l'adresse suivante : https://deepnote.com/
  • Cliquer sur Get started, en haut à droite de l'écran.
  • Si vous possédez un compte Google ou GitHub, cliquer sur Continue with Google ou Continue with Github. Sinon, créer un compte en cliquant sur Sign up with email.
  • Une fois votre compte créé, se connecter à Deepnote.
  • Créer ensuite un nouveau nouveau notebook en cliquant sur Create new, en haut à droite.

4. Interface d'un notebook

Les notebooks doivent être vus comme un outil d'édition de texte avancé. Tout au long de ce cours, nous explorerons ses principales fonctionnalités.

En parcourant l'interface, deux termes assez importants et probablement nouveaux pour vous apparaissent : les cellules (cf. Section 4.1) et les noyaux (cf. Section 4.2). Ces deux notions sont essentielles pour comprendre en quoi un Deepnote est bien plus qu'un simple éditeur de texte.

4.1. Les cellules

Les cellules sont les éléments constitutifs d'un notebook. Il existe deux catégories principales de cellules :

  • Les cellules de code qui contiennent donc du code Python à exécuter dans le noyau (cf. Section 4.2. Lorsque le code est exécuté, le notebook affiche le résultat produit par Python sous la cellule du code correspondant.
  • Une cellule de texte contient du texte formaté à l'aide du langage Markdown et affiche le rendu lorsque la cellule Markdown est exécutée (cf. Aide mémoire Markdown pour notebook).

4.2. Les noyaux

Un noyau est un moteur de calcul qui exécute le code contenu dans un notebook. Derrière chaque notebook, un noyau est démarré attendant patiemment du code à évaluer. Lorsque vous exécutez une cellule de code, ce code est exécuté dans le noyau puis le résultat (ou sortie) est renvoyé à la cellule pour y être affiché. L'état du noyau persiste dans le temps et entre les cellules. Le noyau concerne donc le notebook dans son ensemble et non les cellules individuellement.

Par exemple, si vous importez des bibliothèques ou déclarez des variables dans une cellule, elles seront disponibles dans les autres cellules de votre notebook.

5. Quelques bonnes pratiques rédactionnelles

Un notebook est un excellent support pour réaliser un rapport technique présentant un travail scientifique. Dans ce cas, le notebook doit se plier aux mêmes exigences rédactionnelles.

Voici une liste (non exhaustive) des bonnes pratiques à mettre en oeuvre lors de la rédaction d'un tel notebook :

  • Mettre un titre à votre document permettant d'avoir une idée claire de votre travail.
  • Structurer votre document avec des sections et sous-sections (cf. guide Markdown avec Google Colab).
  • Présenter les données en utilisant des noms usuels pour les variables. Ne pas parler de la variable ID_CODE_ACH mais plutôt de la variable "code achat" ou du "code achat" dans les cellules de texte du notebook. Ne pas hésiter à fournir une table de correspondance entre nom informatique de la variable, nom utilisé dans le notebook et description de la variable.
  • Mettre un tableau de lexique au début du notebook si vous utilisez des acronymes. Toujours définir un acronyme dans le corps du texte à sa première utilisation. Ne pas utiliser de sigle si le terme est peu utilisé dans le notebook.
  • Ne pas surcharger le notebook de statistiques descriptives. Sélectionner les informations importantes et utiles permettant de comprendre votre travail. Mettre les analyses complémentaires réalisées en fin de notebook (créer une section Annexes).
  • Les figures et tableaux doivent être lisibles.
  • Pour chaque tableau de données ou graphique, proposer systématiquement une ou deux aides de lecture, exemple : Dans le tableau 1, la seconde ligne correspond à un individu possédant les caractéristiques A, B, C avec un revenu annuel de X €.
  • Faire obligatoirement référence dans le texte aux figures et tableaux que vous choisissez de présenter.
  • Faire obligatoirement référence dans le texte à la bibliographie présentée dans le rapport.
  • Faire relire le rapport par un tiers (si possible candide et ayant un niveau en français écrit correct).
  • Ne pas hésiter à parcourir le Petit typographe rationnel d'Éddie Saudrais pour faire de jolis documents techniques.
  • Adopter un style concis, factuel et professionnel. Ne surtout pas utiliser un style de rédaction subjectif :
    • Le "je" est à éviter, par exemple : "je commence par ce type d'analyse […], puis j'essaierai la modélisation […]" ;
    • Privilégier la voix impersonnelle, par exemple : "L'analyse […] a été mise en place. Par la suite, la modélisation […] a été réalisée" ;

Remarque finale : Les blocs de code dans un notebook d'étude ne doivent finalement servir qu'à la reproductibilité de votre travail. Au moment de la diffusion de votre étude, vous ne communiquerez in fine qu'un rapport ne contenant que les blocs de texte et les sorties Python du notebook sous-jacent. Il est donc impératif que votre notebook puisse rester lisible et cohérent sans les blocs de code.
