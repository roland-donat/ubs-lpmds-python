1 Objectif

Cette activité a pour objectif de faire une mise à niveau sur la programmation en Python en résolvant une série d'exercices illustrant les concepts de base du langage.

Pour réaliser cette activité, créez un notebook dans le lequel vous allez écrire, exécuter et documenter votre code Python.

Note importante :

Prenez le temps de bien présenter et bien rédiger vos notebooks. Rappel : vous devez considérer un notebook comme un rapport technique susceptible d'être communiqué à des tiers.

2 Exercice : Conversion des durées

Thème: Variables, opérations arithmétiques

Supposons que nous disposons d'une variable dont la valeur est un nombre de secondes : 

nombre_secondes = 3569

Écrivez un code Python permettant de convertir ce nombre de secondes en jours, heures, minutes et secondes et d'afficher le résultat. Vous devez bien sûr utiliser la variable nombre_secondes dans vos calculs.

Aides: Pensez aux opérateurs // et % et à la fonction print.

3 Exercice : indexation des chaînes de caractères

Thème: Chaînes de caractères, indexation

Considérons la chaîne de caractères suivante : 

joueur_basket = "Kawhi Leonard, N°2"
  • Utiliser l'indexation pour construire la chaîne "KLN2".
  • Faire de même en utilisant une indexation négative.
  • Utiliser une opération de slicing pour récupérer la chaîne "Leonard" et la chaîne "Load °".

4 Exercice : Formatage des chaînes de caractères

Thème : Format des chaînes de caractères.

Considérons les variables suivantes : 

num = 2
ts = 52312.5764
owner = 655
occ_pct = 0.001
size = 154232332
  • Utiliser la méthode .format afin de produire la chaîne suivante :
"fichier_002 :  52312.58, 655,   0.1%, 1.54e+05"
  • Même question en utilisant la technique des f-string à la place de la méthode .format.

Aide : Consultez la documentation officielle, en particulier la section 6.1.3.

5 Exercice : The Classic Fizz Buzz Problem

Thème : Structures de contrôle.

Le Fizz Buzz est un problème classique en informatique. Ce dernier est souvent utilisé pour tester des candidats lors d'un entretien d'embauche.

La légende raconte que de nombreuses personnes échouent à ce test lors des entretiens.

Objectif : Écrire un programme qui affiche les nombres de 1 à 100 mais :

  • pour les multiples de 3, affiche Fizz à la place du nombre.
  • pour les multiples de 5, affiche Buzz à la place du nombre.
  • Pour les multiples de 3 et 5, affiche FizzBuzz à la place du nombre.

Aide : Penser à l'opérateur %.

6 Exercice : Traitement de données textuelles

Source : exercice adapté à partir de PythonCert.

Thème : fichiers, structures de données

Télécharger le fichier students.txt dans votre espace de travail.

Ce fichier contient une liste de personnes avec, pour chacune d'elles, les langages de programmations maîtrisés.

Le format du fichier est le suivant :

  • la première ligne du fichier est une entête rappelant le format ;
  • chaque ligne est construite comme suit : Lastname, Firstname: Nickname, languages ;
  • les éléments Lastname et Firstname sont toujours présents et sont séparés par une virgule et un espace ;
  • après le Firstname, on trouve le caractère : ;
  • le Nickname commence toujours par une majuscule ;
  • les langages commencent toujours par une minuscule ;
  • chaque personne peut avoir un Nickname ou aucun ;
  • chaque personne peut maîtriser zéro ou un ou plusieurs langages.

L'objectif général de l'exercice est de parcourir le fichier afin d'extraire les informations de chaque personne en les organisant dans une structure de données adéquates.

Travail à réaliser : #. Ouvrir le fichier en lecture. Aide : fonction open et structure with ... as. #. Extraire pour chaque personne, son nom, prénom, surnom (si présent) et liste des langages (si présente). Aide : méthodes .split, .trim #. Stocker ces éléments dans une structure adaptée. #. À partir de la structure construite à l'étape précédente :

Jagger, Michael: Mick, shell, python

So a colon after the name, then the nickname, and then one or more languages.

However, like real data files, the file is NOT well-formed. Only some lines have nicknames, and other small differences, so you will need to write some code to make sure you get it all correct.

Write a little script that reads that file and generates a list of all the languages that have been used.

Aide :

What might be the best data structure to use to keep track of bunch of values (the languages) without duplication?

The file format: The first line of the file is:

Name: Nickname, languages

And each line looks something like this:

Jagger, Michael: Mick, shell, python

So a colon after the name, then the nickname, and then one or more languages.

However, like real data files, the file is NOT well-formed. Only some lines have nicknames, and other small differences, so you will need to write some code to make sure you get it all correct.

How can you tell the difference between a nickname and a language?

Extra challenge: keep track of how many students specified each language.
